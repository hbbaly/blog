理解 Object.defineProperty 及实现数据双向绑定
发表于 2018-07-20| 更新于 2019-10-23|js

概述

Object.defineProperty() 和 Proxy 对象，都可以用来对数据的劫持操作。何为数据劫持呢？就是在我们访问或者修改某个对象的某个属性的时候，通过一段代码进行拦截行为，然后进行额外的操作，然后返回结果。那么vue中双向数据绑定就是一个典型的应用。

Object.defineProperty的语法

  • obj: 可以理解为目标对象。
  • prop: 目标对象的属性名。
  • descriptor: 对属性的描述。

数据描述符

数据描述符有如上configurable，enumerable，value 及 writable 配置项。

value

属性对应的值，值的类型可以是任意类型的。

writable

属性的值是否可以被重写，设置为true可以被重写，设置为false，是不能被重写的，默认为false。

writable 默认为false，不能被重写，但是我们将它设置为true，就可以进行重写值了.

enumerable

是否可以被枚举，比如使用 for..in 或 Object.keys() 这样的。设置为true可以被枚举，设置为false，不能被枚举，默认为false

configurable

可配置的意思，那么该属性的含义是：是否可以删除目标属性。如果我们设置它为true的话，是可以被删除。如果设置为false的话，是不能被删除的。它默认值为false。

使用 delete命令删除 obj.name的话，该属性值是删除不了的，因为 configurable 默认为false，不能被删除的。
但是如果我们把它设置为true，那么就可以进行删除了。

对数组的监听

当我们使用 Object.defineProperty 对数组赋值有一个新对象的时候，会执行set方法，但是当我们改变数组中的某一项值的时候，或者使用数组中的push等其他的方法，或者改变数组的长度，都不会执行set方法。

也就是如果我们对数组中的内部属性值更改的话，都不会触发set方法。因此如果我们想实现数据双向绑定的话，我们就不能简单地使用 obj.name[1] = newValue； 这样的来进行赋值了。那么对于vue这样的框架，那么一般会重写 Array.property.push方法，并且生成一个新的数组赋值给数据，这样数据双向绑定就触发了。
